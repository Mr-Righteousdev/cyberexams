<objective>
Add success/error toast notifications for all admin CRUD actions, and add loading states to all Livewire component forms. Every create, update, publish, deactivate, reactivate, and delete action must show a Flux toast. Every form submission must show a loading state.

Purpose: Users don't know if their action succeeded without feedback. Loading states prevent double-submit during slow operations.

Output: Flash messages on all Livewire actions, loading states on all forms.
</objective>

  <files>resources/views/livewire/admin/student-create.blade.php, resources/views/livewire/admin/exam-create.blade.php, resources/views/livewire/admin/question-create.blade.php, resources/views/livewire/admin/manual-grading.blade.php, resources/views/livewire/admin/exam-index.blade.php, resources/views/livewire/admin/student-index.blade.php</files>
  <action>
For each admin form view, add wire:loading indicator to the submit/primary action button and wire:disabled to prevent double-submit during the request.

**student-create.blade.php:** On the primary submit <flux:button>, add:
```html
wire:loading.attr="disabled"
wire:loading.class.add("opacity-50 cursor-not-allowed")
```
And add a loading indicator below or inside the button:
```html
<flux:button type="submit" wire:loading.attr="disabled" wire:loading.class.add("opacity-50 cursor-not-allowed")>
    <span wire:loading>Creating...</span>
    <span wire:target="createStudent">Create Student</span>
</flux:button>
```

**exam-create.blade.php:** Same pattern on the save/publish button with "Saving..." / "Save Exam" text swap.

**question-create.blade.php:** Same pattern on the save button with "Adding..." / "Add Question".

**manual-grading.blade.php:** Same pattern on the save grade button with "Saving..." / "Save Grade".

**exam-index.blade.php:** On delete buttons, add confirmation via a simple wire:confirm or a flux:dialog. Do NOT add loading states to delete buttons (they should be instant). Add a simple confirmation: `x-on:click.prevent="$confirm('Delete this exam?')"`.

**student-index.blade.php:** On deactivate/reactivate buttons, add a `wire:loading` indicator or icon.

Read each view file first to find the exact submit button structure. Only modify the submit buttons — do not add loading to navigation or non-form actions.
