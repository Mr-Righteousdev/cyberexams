<task type="auto">
  <name>Task 2: Create Dashboard Livewire component</name>
  <files>app/Livewire/Student/Dashboard.php, resources/views/livewire/student/dashboard.blade.php</files>
  <action>
Create `app/Livewire/Student/Dashboard.php` with the following logic:

**Properties:** None needed — load directly in `mount()`.

**mount():** Query available exams:
```php
$examIds = Auth::user()->examSessions()->pluck('exam_id');
$exams = Exam::where('is_published', true)
    ->whereNotIn('id', $examIds)  // exclude exams already attempted
    ->where(function ($q) {
        $q->whereNull('starts_at')->orWhere('starts_at', '<=', now());
    })
    ->where(function ($q) {
        $q->whereNull('ends_at')->orWhere('ends_at', '>=', now());
    })
    ->withCount('questions')
    ->get();
```

**Additional queries:**
- `$attempted = Auth::user()->examSessions()->with('exam')->get()` — for "already submitted" cards
- Do NOT show submitted exams as "startable"

**render():** Pass `$exams` and `$attempted` to the view.

Create `resources/views/livewire/student/dashboard.blade.php`:
- Loop over `$exams` — show exam cards with title, description, duration, question count, status badge "Available"
- Loop over `$attempted` — show cards with status "Submitted", a "View Results" button linking to `route('student.results', $session)`
- If no exams and no attempts: show empty state "No exams available at this time."
- Use `<flux:card>` for each exam card
- Use `<flux:badge variant="success">` for Available, `<flux:badge variant="neutral">` for Submitted
- Wire up the student layout via component class: set `$layout` property or use `@extends('layouts.student')` in the Blade view
- "Start Exam" button links to `route('student.exam.start', $exam)` — create the route in the next task
  </action>
  <verify>Student sees published exams, submitted exams show results link, no unpublished exams visible</verify>
