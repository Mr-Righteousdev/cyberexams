Add server-side IP change detection to ExamTaking component.

**Modify existing IP check in mount():**
The current mount has:
```php
// IP check (SESS-03)
if ($session->ip_address !== request()->ip()) {
    abort(403, 'IP address mismatch. Exam session is locked.');
}
```

Expand to track IP changes and flag the session:
```php
// IP check with change detection (GUARD-06)
$currentIp = request()->ip();
if ($session->ip_address !== $currentIp) {
    // IP has changed - log and flag
    ActivityLog::create([
        'session_id' => $session->id,
        'event_type' => 'ip_change',
        'metadata' => [
            'original_ip' => $session->ip_address,
            'new_ip' => $currentIp,
            'changed_at' => now()->toIso8601String(),
        ],
        'occurred_at' => now(),
    ]);
    
    // Flag session
    $session->update([
        'is_flagged' => true,
        'flag_reason' => 'IP address changed during exam: ' . $session->ip_address . ' -> ' . $currentIp,
    ]);
    
    // Optionally block access completely or allow with warning
    // Option 1: Block entirely (strict)
    abort(403, 'IP address changed during exam. Session has been flagged.');
    
    // Option 2: Allow with warning (lenient - use this for flexibility)
    // Remove abort, continue with flagged status shown to admin
}
```

**Decision for this implementation:**
Use option 2 (lenient) - flag and log but allow continuation. This is better UX:
1. Don't abort
2. Log the change
3. Flag the session
4. Show warning to student (optional)

**Update the code:**
```php
// IP check with change detection (GUARD-06)
$currentIp = request()->ip();
if ($session->ip_address !== $currentIp) {
    // Store original IP to compare
    $originalIp = $session->ip_address;
    
    // Log and flag
    ActivityLog::create([
        'session_id' => $session->id,
        'event_type' => 'ip_change',
        'metadata' => [
            'original_ip' => $originalIp,
            'new_ip' => $currentIp,
            'changed_at' => now()->toIso8601String(),
        ],
        'occurred_at' => now(),
    ]);
    
    // Flag but don't block
    $session->update([
        'is_flagged' => true,
        'flag_reason' => 'IP address changed: ' . $originalIp . ' -> ' . $currentIp,
    ]);
    
    // Optionally dispatch warning (student sees but can continue)
    $this->dispatch('notify', [
        'message' => 'Warning: Network changed during exam. Session has been flagged.',
        'type' => 'warning'
    ]);
}
```
