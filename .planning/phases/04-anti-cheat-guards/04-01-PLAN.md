  <name>Task 1: Add Anti-Cheat event guards to exam-taking view</name>
  <files>resources/views/livewire/student/exam-taking.blade.php</files>
  <action>
Add anti-cheat event handlers to the exam-taking view. These guards prevent common cheating attempts via browser interactions.

**Add to the root div (`<div class="mx-auto max-w-4xl">`):**
Add the following Alpine.js x-data and event handlers:
```html
<div class="mx-auto max-w-4xl"
     x-data="{ 
         init() {
             // Block copy
             document.addEventListener('copy', (e) => {
                 e.preventDefault();
                 this.showBlockedMessage('Copy is disabled during exam');
             });
             // Block paste
             document.addEventListener('paste', (e) => {
                 e.preventDefault();
                 this.showBlockedMessage('Paste is disabled during exam');
             });
             // Block cut
             document.addEventListener('cut', (e) => {
                 e.preventDefault();
                 this.showBlockedMessage('Cut is disabled during exam');
             });
             // Block context menu (right-click)
             document.addEventListener('contextmenu', (e) => {
                 e.preventDefault();
                 this.showBlockedMessage('Right-click is disabled during exam');
             });
             // Block keyboard shortcuts
             document.addEventListener('keydown', (e) => {
                 // Block Ctrl+U (view source)
                 if (e.ctrlKey && e.key === 'u') {
                     e.preventDefault();
                     this.showBlockedMessage('Ctrl+U is disabled');
                 }
                 // Block Ctrl+S (save)
                 if (e.ctrlKey && e.key === 's') {
                     e.preventDefault();
                     this.showBlockedMessage('Ctrl+S is disabled');
                 }
                 // Block Ctrl+P (print)
                 if (e.ctrlKey && e.key === 'p') {
                     e.preventDefault();
                     this.showBlockedMessage('Ctrl+P is disabled');
                 }
                 // Block Ctrl+Shift+I (dev tools)
                 if (e.ctrlKey && e.shiftKey && e.key === 'I') {
                     e.preventDefault();
                     this.showBlockedMessage('Developer tools are disabled');
                 }
                 // Block F12
                 if (e.key === 'F12') {
                     e.preventDefault();
                     this.showBlockedMessage('F12 is disabled');
                 }
                 // Block Escape in exam
                 if (e.key === 'Escape') {
                     e.preventDefault();
                 }
             });
         },
         showBlockedMessage(msg) {
             alert('Blocked: ' + msg);
         }
     }"
     x-init="init()">
```

**Simpler approach - inline handlers:**
Add directly to the root element using Alpine inline handlers:
```html
<div class="mx-auto max-w-4xl"
     x-data
     x-on:copy.prevent.window="$flux.toast({variant: 'warning', text: 'Copy is disabled during exam'}); return false;"
     x-on:paste.prevent.window="$flux.toast({variant: 'warning', text: 'Paste is disabled during exam'}); return false;"
     x-on:cut.prevent.window="$flux.toast({variant: 'warning', text: 'Cut is disabled during exam'}); return false;"
     x-on:contextmenu.prevent.window="$flux.toast({variant: 'warning', text: 'Right-click is disabled'}); return false;"
     x-on:keydown.window.debounce.50ms="
         if ($event.ctrlKey && ['u','s','p'].includes($event.key.toLowerCase())) {
             $flux.toast({variant: 'warning', text: 'Ctrl+'+$event.key.toUpperCase()+' is disabled'});
             $event.preventDefault();
         }
         if ($event.key === 'F12') {
             $flux.toast({variant: 'warning', text: 'F12 is disabled'});
             $event.preventDefault();
         }
     ">
```

Note: Flux toast may not support object syntax - use the simpler pattern. For safety, use a hidden toast element triggered via Livewire:

1. Add to ExamTaking component:
```php
public string $lastBlockedMessage = '';

public function blockMessage(string $message): void
{
    $this->lastBlockedMessage = $message;
    $this->dispatch('notify', ['message' => $message, 'type' => 'warning']);
}
```

2. Add to Blade:
```html
<div class="mx-auto max-w-4xl"
     x-data
     x-on:copy.prevent.window="$dispatch('block-message', {message: 'Copy is disabled'})"
     x-on:paste.prevent.window="$dispatch('block-message', {message: 'Paste is disabled'})"

             $dispatch('block-message', {message: 'Ctrl+'.$event.key.toUpperCase()+' is disabled'});
             $event.preventDefault();
         }
         if ($event.key === 'F12') {
             $dispatch('block-message', {message: 'F12 is disabled'});
             $event.preventDefault();
         }
     ">
```

**For question content specifically:**
Wrap the question text in a div with these classes to ensure users cannot select text but can still interact with form inputs:

Look for the question display section and ensure the text area and radio buttons are still clickable. The `select-none` on the parent will prevent selection everywhere. Instead, apply it more surgically:

**Targeted approach:**

```html

```

And for the options - they MUST remain selectable for radio/button interaction, but add pointer-events and cursor styled:
```html
<label class="flex cursor-pointer items-center gap-3 rounded-lg border border-zinc-200 p-4 hover:bg-zinc-50 ...">
```

Actually, for exam content that's being displayed (not input), apply `select-none` to question text only:
- Question text section gets `select-none`

For MCQ/TF/Short Answer questions - question text gets `select-none`:
```html

</pre>
```

Also add to the entire page container to prevent drag selection:
```html
<div class="mx-auto max-w-4xl select-none" ...>
```

This prevents text selection anywhere, which is the standard for locked-down exams.
