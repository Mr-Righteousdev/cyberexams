<task type="auto">
  <name>Task 2: Create Exam Data Migrations</name>
  <files>database/migrations</files>
  <action>
Create the following migrations in this order:

1. exams table:
   - id (bigint, PK)
   - title (string, 255)
   - description (text, nullable)
   - duration minutes (integer)
   - instructions (text, nullable)
   - starts_at (timestamp, nullable)
   - ends_at (timestamp, nullable)
   - shuffle_questions (boolean default false)
   - shuffle_options (boolean default false)
   - is_published (boolean default false)
   - passing_marks (decimal, default 0)
   - admin_id (bigint, FK to users)
   - timestamps

2. questions table:
   - id (bigint, PK)
   - exam_id (bigint, FK to exams)
   - type (enum: mcq, true_false, short_answer, code_snippet)
   - question_text (text)
   - marks (decimal, default 1)
   - order (integer)
   - code_language (string, nullable - for code_snippet)
   - timestamps

3. options table:
   - id (bigint, PK)
   - question_id (bigint, FK to questions)
   - option_text (string)
   - is_correct (boolean default false)
   - order (integer)
   - timestamps

4. exam_sessions table:
   - id (bigint, PK)
   - exam_id (bigint, FK to exams)
   - user_id (bigint, FK to users)
   - ip_address (string, 45)
   - started_at (timestamp)
   - submitted_at (timestamp, nullable)
   - is_flagged (boolean default false)
   - flag_reason (text, nullable)
   - timestamps

5. answers table:
   - id (bigint, PK)
   - exam_session_id (bigint, FK to exam_sessions)
   - question_id (bigint, FK to questions)
   - answer_text (text, nullable)
   - selected_option_id (bigint, nullable, FK to options)
   - is_correct (boolean, nullable - auto-graded)
   - marks_awarded (decimal, nullable - manual)
   - timestamps

6. activity_logs table:
   - id (bigint, PK)
   - exam_session_id (bigint, FK to exam_sessions, nullable)
   - event_type (string)
   - event_data (json, nullable)
   - ip_address (string, nullable)
   - timestamps
  </action>
